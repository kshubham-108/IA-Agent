PROBLEM or Opportunity Statement
This Vision Document has been updated to combine 
4
 separate pieces of work, with the desire that all 
4
 pieces of work would be managed by 1 x project manager concurrently.  The 
4
 projects are not in any way dependent on each other or related in any way.
Project 1
:
Ab
z
orb
 (SPID 536) requires the creation of a new ABS SPIM SSH user in addition to the existing user (SPIM_536_01). The new user should have the same roles and permissions as the current user and must be accessible via the existing VPN setup.
This is needed as Absorb has recently onboarded 
Wavenet
 onto their billing system and needs to provide separate access to 
Wavenet
 agents for BAU activities. Since the current user does not support concurrent logins, an additional user is required to allow multiple users to access the system at the same time.
Project 2: 
DBFB 
(SPID 570) 
require 4 new ABS SPIM SSH 
users;
1 login for provisioning
1 login for network status checking
1 login for IMEI checking
1 login for bars
This is to help them manage their 
speed of transactions on ABS SPIM which are very slow 
at the moment
.

Wavenet
 
(SPID 370) 
require a
n MFT area setting up specifically for them.  
This is for sharing commercial month end reporting that shouldn't go to 
Abzorb
 (their parent)
.  Daily CDR's would stay as they are.
Project 4:
: G
ia
com wants to move the termination point of an existing VPN tunnel to a new virtual router within their network for better segregation.
Technical Details
: 
Current VPN terminates on Palo Alto firewall; new termination will be on a different VR with new IP.
O2 policy requires creating a 
new VPN
 if source IP changes.
Ports currently allowed need to be confirmed and replicated for the new VPN.
Historical Context
: 
Previous VPN setup was done in 2022; similar process expected now.
Past delays were due to missed firewall configurations; aim to avoid this now.
Out-of-Scope Items
: 
MFT and SOAP APIs confirmed as unrelated to this VPN change.
Dependencies
: 
O2 internal approval process for VPN setup.
