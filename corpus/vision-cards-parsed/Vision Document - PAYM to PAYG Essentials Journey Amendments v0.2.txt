PROBLEM or Opportunity Statement
The core issue is that there is no simple, seamless way to move from PAYM to PAYG
 Essentials, or vice versa, 
using the same SIM, resulting in unnecessary effort, delays, and a poor customer experience.
The current process of disconnecting and reconnecting customers is a 
manual back-office process
, which can be impacted by human error, so f
ailure of any steps can lead customers without service.

SCOPE
 
5.1 
In Scope
Tesco Mobile is reviewing the PAYM to PAYG Essential customer journey. Currently, moving customers between these segments (PAYM to PAYG and vice versa) relies on a highly manual process.
To improve this, Tesco Mobile needs to enhance capabilities within NetCracker to support re-provisioning of a recently disconnected mobile number (MPN) on the same SIM. 
This would allow the number to be set up under a different BSS account and assigned a new service more efficiently.
OUT OF Scope
There is no change required for other customer journeys.

Tesco Mobile provisions Mobile Phone Numbers (MPNs) via the Network Core (NC) API by passing:
MPN
ICCID ( optional )
BSS Account Identifier (Related Party ID)
NC uses these inputs to:
Provision the MPN across network elements
Create and maintain service inventory
Associate the MPN/SIM with a service instance ID and BSS account
Subsequent lifecycle operations (e.g., SIM swap) rely on:
BSS Account Identifier
Service Instance ID
When a customer transitions between propositions (e.g., PAYG → PAYM or PAYM → PAYG):
The MPN is disconnected
A new provisioning request is triggered under a different BSS account
However:
NC enforces lifecycle rules that prevent reuse of a disconnected MPN
As a result, the same MPN + SIM combination cannot be reprovisioned immediately
We need the capability of re-provisioning of a recently disconnected MPN on the same SIM, under a different BSS account and with different service.
